# 区域录制实时标注设计规格

## 目标

在现有区域录制流程中增加实时标注。用户开始录制后，可以在“指针”和“标注”模式之间切换，使用画笔、矩形、椭圆和箭头在选区内绘制。标注从绘制时刻开始进入视频，并持续保留，直到用户撤销或清除。

本次保持现有录制约束不变：仅导出 H.264 MP4、无音频、停止后先预览再保存。

## 已确认的产品决策

- 首期实现基础标注：画笔、矩形、椭圆、箭头、颜色、粗细、撤销和清除。
- 标注从绘制时刻开始显示，并持续到撤销或清除。
- 默认使用指针模式，选区内的鼠标可以操作底层应用。
- 选择任一标注工具后，标注层拦截选区内的鼠标用于绘制。
- 暂停录制时仍可标注；恢复后的第一帧直接显示暂停期间整理后的最终状态。
- 标注工具扩展到现有录制控制条，不创建独立工具栏。
- 使用受内容保护的透明标注层显示交互预览，再把相同标注显式合成到录制 Canvas。
- 重新录制时清空全部标注，不沿用上一段录制内容。

## 用户流程

1. 用户完成截图选区并进入区域录制，现有蓝色边框和录制控制条出现。
2. 待录制和倒计时阶段默认处于指针模式，标注工具不可用。
3. 倒计时结束后，标注工具启用，录制边框变红。
4. 用户保持指针模式时，可以正常操作选区内的底层应用。
5. 用户选择画笔、矩形、椭圆或箭头后，选区内的拖动操作用于绘制。
6. 绘制中的临时笔迹实时显示，也实时合成进后续视频帧；松开鼠标后成为持久标注。
7. 用户可以改变后续标注的颜色和粗细，也可以撤销最后一条标注或清空全部标注。
8. 用户切回指针模式后，已有标注继续显示，但鼠标重新穿透到底层应用。
9. 暂停期间可以继续绘制、撤销或清除。恢复录制时，视频从当前最终标注状态继续。
10. 停止录制后隐藏标注层，现有预览和 MP4 保存流程保持不变。
11. 重新录制先清空标注，再沿用同一选区进入倒计时。

## 界面设计

### 录制控制条

在现有状态、FPS 和录制控制旁增加标注组：

- 指针
- 画笔
- 矩形
- 椭圆
- 箭头
- 颜色与粗细
- 撤销
- 清除

按钮使用与截图工具栏一致的图标风格，并提供中文 `title` 和 `aria-label`。当前工具显示蓝色选中态。待录制、倒计时、停止和预览阶段禁用标注组；录制中和暂停时启用。

默认颜色为红色，默认粗细为 4px。颜色面板提供红、橙、黄、绿、蓝、黑、白，粗细提供 2、4、7px。切换工具、颜色或粗细不会修改已有标注。撤销在没有标注时禁用；清除立即移除全部标注，不弹出确认框。

控制窗口优先使用单行布局，目标宽度不超过 760px，并始终限制在当前显示器工作区内。当可用宽度不足 700px 时切换为两行：第一行保留状态、计时和录制控制，第二行显示标注工具。样式弹出面板根据工作区边界向上或向下展开。

### 选区标注层

复用现有 `recordFrameWindow`。窗口仍比选区四边各向外扩展 2px，用于显示不入镜的状态边框；标注 Canvas 只覆盖内部选区，不绘制到外侧边框区域。

- 指针模式调用 `setIgnoreMouseEvents(true)`，窗口不抢占底层应用输入。
- 标注模式调用 `setIgnoreMouseEvents(false)`，窗口接收选区内的指针事件。
- 窗口继续启用 `setContentProtection(true)`，透明标注层本身不会被桌面采集直接录入。
- 停止、取消、预览或异常降级时恢复鼠标穿透，避免留下无法操作的透明窗口。

## 架构

### 组件边界

1. `record/record.html`、`record/record.css` 与 `record/record.js`
   - 扩展录制控制条和响应式布局。
   - 管理当前工具、颜色、粗细、撤销和清除命令。
   - 接收标注窗口发来的最新标注快照。
   - 每帧先绘制桌面裁剪画面，再绘制活动标注和持久标注。

2. `record/frame.html` 与新增的标注脚本、样式
   - 保留蓝、红、黄边框状态。
   - 在选区内部渲染透明交互 Canvas。
   - 接收工具和样式状态，采集指针事件并构建标注。
   - 在绘制过程中发送受节流的实时快照，完成时发送最终快照。

3. `record/annotation-utils.js`
   - 提供可在渲染进程和 Node 测试中复用的纯逻辑。
   - 校验工具、颜色、粗细和标注数据。
   - 构建画笔点集和形状几何数据。
   - 提供撤销、清除、坐标缩放和 Canvas 绘制函数。
   - 不依赖 Electron、DOM 或文件系统。

4. `preload-record.js` 与 `preload-record-frame.js`
   - 只暴露有限的标注命令、初始化和快照事件。
   - 不暴露任意 IPC、Node.js 或文件系统能力。

5. `main.js`
   - 复用现有控制窗口和边框窗口生命周期。
   - 在两个窗口之间验证并转发工具状态、输入模式和标注快照。
   - 根据控制条响应式尺寸计算窗口位置。
   - 在停止、取消、重录和异常关闭时恢复鼠标穿透并清理状态。

### 标注数据模型

所有坐标均为相对选区左上角的逻辑像素，不能保存全局屏幕坐标。

```js
{
  id: 1,
  type: 'pen' | 'rect' | 'ellipse' | 'arrow',
  color: '#ff3b30',
  width: 4,
  x: 20,
  y: 30,
  x2: 160,
  y2: 120,
  points: [{ x: 20, y: 30 }, { x: 22, y: 33 }]
}
```

`points` 只用于画笔，其他工具使用起点和终点。矩形和椭圆允许从任意方向拖动，绘制时统一规范化边界。箭头头部尺寸根据线宽计算，并设置合理的最小值。

为避免异常输入造成内存持续增长，单次录制最多保留 500 条标注，单条画笔最多保留 10000 个采样点。画笔移动事件按动画帧合并相邻采样，并忽略不足 0.5 逻辑像素的重复点。

### 数据流

```text
控制条选择工具/样式
  -> 主进程验证录制控制窗口
  -> 转发给 recordFrameWindow
  -> 主进程切换 frameWindow 鼠标穿透

选区内 pointerdown / pointermove / pointerup
  -> 标注窗口构建活动标注
  -> 本地透明 Canvas 实时绘制
  -> 发送最新快照给主进程
  -> 主进程验证、限制大小并转发给录制窗口
  -> 录制窗口在下一次绘制循环合成标注
  -> canvas.captureStream(frameRate)
  -> MediaRecorder / WebM 临时文件 / H.264 MP4
```

主进程只接受当前 `recordWindow` 和其对应 `recordFrameWindow` 的消息。标注快照只允许约定字段、有限工具类型、合法十六进制颜色、白名单粗细和有限数组长度，不允许传递 HTML、图片数据、文件路径或任意频道名。

## 坐标与绘制

现有桌面流按 `selectionBounds` 裁剪为录制 Canvas。标注坐标使用选区逻辑像素，合成时分别计算：

```text
scaleX = recordingCanvas.width / selectionBounds.width
scaleY = recordingCanvas.height / selectionBounds.height
```

点坐标分别乘以 `scaleX` 和 `scaleY`。线宽使用两者的几何平均值缩放，避免高 DPI 或偶数尺寸修正造成明显粗细偏差。所有工具使用圆角端点和圆角连接；形状绘制裁剪在录制 Canvas 范围内。

透明标注 Canvas 的原点位于边框窗口内部 `(2, 2)`，其 CSS 尺寸严格等于选区逻辑尺寸。边框、控制条、工具选中态和样式面板不参与录制 Canvas 合成，因此不会进入最终视频。

## 状态与生命周期

### 工具状态

```text
idle/countdown -> pointer only, annotation controls disabled
recording      -> pointer or annotation tools enabled
paused         -> pointer or annotation tools enabled
preview/saving -> annotation layer hidden, pointer passthrough restored
```

工具状态不改变现有录制状态机。暂停只暂停 `MediaRecorder` 和计时，不暂停标注模型。暂停期间产生的中间绘制过程不会进入视频，恢复后的第一帧使用当时的完整快照。

### 标注生命周期

- 活动标注在拖动过程中进入实时快照，因此绘制动作本身可见于视频。
- `pointerup` 后活动标注提交到持久列表。
- 撤销移除最后提交的一条标注，不影响正在绘制的临时标注。
- 清除移除持久列表和活动标注。
- 切换回指针模式时取消未完成的活动标注，但保留已提交内容。
- 停止后冻结最终快照，随后隐藏标注层。
- 重新录制、取消、关闭窗口和新建录制流程都会清空标注状态。

## 错误处理与降级

- 标注窗口未就绪：录制仍可开始，标注按钮保持禁用并显示“标注工具不可用”。
- 标注窗口异常关闭：主进程立即恢复其鼠标穿透状态，录制继续；控制条禁用标注功能。
- 快照校验失败或超过限制：丢弃该快照并记录日志，保留最后一个有效快照，不中断视频。
- Canvas 标注绘制失败：停止合成标注并报告错误，桌面画面继续录制，用户仍可停止和保存。
- IPC 转发失败：控制条回到指针模式，避免透明窗口持续拦截输入。
- 停止、取消和预览转换使用 `finally` 恢复鼠标穿透并隐藏标注层。

## 测试策略

### 自动化测试

- `record/annotation-utils.js`
  - 工具、颜色和粗细白名单校验。
  - 画笔采样去重和点数上限。
  - 任意拖动方向的矩形、椭圆规范化。
  - 箭头几何和最小箭头尺寸。
  - 高 DPI、非整数比例和偶数尺寸修正下的坐标缩放。
  - 撤销、清除、切换指针和重新录制重置。
  - 使用模拟 Canvas context 验证四种工具的绘制调用。

- IPC 与静态契约
  - 控制预加载层只暴露约定标注命令。
  - 标注预加载层只暴露工具事件和快照提交。
  - 主进程校验控制窗口、标注窗口和父子关系。
  - 指针模式启用鼠标穿透，标注模式关闭鼠标穿透。
  - 停止、取消、预览、异常和重录路径恢复鼠标穿透。
  - 录制绘制循环按“桌面画面、标注快照”的顺序合成。

### 手工验证

- 录制中分别使用画笔、矩形、椭圆和箭头。
- 切换七种颜色和三档粗细，确认只影响后续标注。
- 指针模式可以操作底层应用，标注模式只在选区内绘制。
- 录制中撤销和清除后，视频从对应时刻移除标注。
- 暂停时绘制、撤销和清除，恢复后的第一帧显示最终状态。
- 停止后预览、取消保存、重新录制和再次保存。
- 重新录制不带入上一次标注。
- 主显示器、高 DPI 和负坐标副屏。
- 窄工作区下两行控制条和样式面板不超出屏幕。
- portable 构建中导出 MP4，并用视频关键帧确认标注出现时机。
- 使用 `ffprobe` 确认输出仍为 H.264、无音轨且只有 MP4 用户输出。

## 非目标

本次不实现以下内容：

- 文字、序号、马赛克、荧光笔、橡皮擦或标注对象选择编辑。
- 重做、图层排序、移动、缩放或旋转已有标注。
- 标注自动消失、淡入淡出或固定显示时长。
- 鼠标点击动画、光标高亮或键盘提示。
- 在预览页继续编辑标注。
- 音频、GIF、WebM 用户输出或视频剪辑。
- 跨显示器选区和录制过程中调整选区尺寸。

## 验收标准

1. 录制开始后，控制条可以在指针、画笔、矩形、椭圆和箭头之间切换。
2. 指针模式不阻塞底层应用；标注模式可以在选区内稳定绘制。
3. 绘制过程从发生时刻进入视频，完成后持续显示到撤销或清除。
4. 颜色和粗细配置正确应用于新标注，不修改已有标注。
5. 暂停期间可以整理标注，恢复后的第一帧显示暂停结束时的最终状态。
6. 边框、控制条和标注交互窗口本身不进入视频，只有显式合成的标注进入 MP4。
7. 停止、取消、重录和异常路径不会留下拦截鼠标的透明窗口。
8. 重新录制清空上一段标注，现有预览和保存流程保持正常。
9. 高 DPI、负坐标副屏和窄工作区下坐标、布局与最终画面正确。
10. 最终文件仍为无音轨的 H.264 MP4。
