# 区域录制设计规格

## 目标

在截图选区完成后的工具栏中增加“区域录制”入口。用户可以对当前选区进行无声录制，经过 3 秒倒计时后开始，支持暂停、继续、停止和取消。停止后先预览，再保存为 H.264 MP4。

该功能参考 PixPin 的屏幕录制交互，但本次只实现基础闭环：固定选区、无声音、无剪辑、仅 MP4。

## 已确认的产品决策

- 入口位于截图选区工具栏中。
- 输出格式仅支持 MP4；内部临时格式不向用户展示。
- 不录制麦克风或系统声音。
- 停止后进入预览，再由用户决定保存、重新录制或关闭。
- 预览只支持播放检查、进度拖动、保存和重新录制，不支持裁剪、倍速或删除片段。
- 点击开始后倒计时 3 秒。
- 待录制边框为蓝色，录制中为红色，暂停时为黄色。
- 使用 Canvas 裁剪桌面流、MediaRecorder 生成临时 WebM，再由随应用打包的 FFmpeg 转码为 MP4。

## 用户流程

1. 用户启动截图并完成选区。
2. 截图工具栏显示“区域录制”按钮。
3. 用户点击按钮，截图窗口提交当前选区的屏幕坐标、尺寸和显示器信息，然后关闭。
4. 应用在原选区显示蓝色边框，并在选区外侧显示录制控制条。
5. 用户点击开始，控制条显示 3、2、1；倒计时期间可以取消。
6. 倒计时结束后开始录制，边框变红，计时从 00:00 开始。
7. 用户可以暂停、继续、停止或取消。暂停时边框变黄，暂停时间不计入录制时长。
8. 用户点击停止后，录制流停止，临时 WebM 文件完成写入，控制窗口切换为预览页。
9. 用户可以播放、拖动进度、保存 MP4、重新录制或关闭。
10. 保存时先选择目标路径，再显示转码进度。成功后显示保存路径并结束录制流程。

## 界面设计

### 截图工具栏

在“识别并翻译”和“固定到屏幕”之间增加一个录制按钮。按钮使用熟悉的录制圆点图标，`title` 为“区域录制”。选区宽或高不足 16 个逻辑像素时按钮保持禁用。

点击录制按钮不会导出截图，也不会把截图标注内容录入视频。录制源始终是选区对应的实时桌面画面。

### 选区边框

主进程创建透明、置顶、鼠标穿透的边框窗口，边框窗口覆盖选区但不接收输入。窗口启用内容保护，确保边框和倒计时不会进入桌面采集结果。

- 待录制：蓝色边框。
- 录制中：红色边框。
- 暂停：黄色边框。
- 预览、取消或结束：隐藏边框。

选区在进入录制流程后固定，不支持移动或缩放。重新录制沿用同一选区。

### 录制控制条

控制条位于选区下方；空间不足时放在选区上方，并限制在当前显示器工作区内。控制条启用内容保护，不会进入录制画面。

状态与控件如下：

| 状态 | 显示内容 | 可用操作 |
| --- | --- | --- |
| 待录制 | `00:00`、FPS、开始、关闭 | 开始、关闭 |
| 倒计时 | `3 / 2 / 1`、取消 | 取消 |
| 录制中 | 已录时长、FPS、暂停、停止、取消 | 暂停、停止、取消 |
| 暂停 | 已录时长、FPS、继续、停止、取消 | 继续、停止、取消 |
| 保存中 | 转码进度、取消禁用提示 | 等待完成 |

### 预览页

停止后复用录制控制窗口并调整为居中的预览窗口。页面包含视频画面、播放/暂停、原生进度拖动、当前时间/总时长、保存 MP4、重新录制和关闭。

预览页不提供首尾裁剪、片段删除、播放倍速、鼠标效果、键盘效果或标注编辑。

## 架构

### 组件边界

1. `capture/capture.html` 与 `capture/capture.js`
   - 展示区域录制按钮。
   - 校验最小选区。
   - 把选区屏幕坐标传给预加载桥接层。

2. `preload-capture.js`
   - 暴露 `startRegionRecording(selectionBounds)`。
   - 只允许发送数值化后的矩形数据。

3. `main.js`
   - 验证请求来自当前截图窗口。
   - 根据选区定位显示器并解析桌面采集 source ID。
   - 关闭截图窗口，创建录制控制窗口和边框窗口。
   - 注册录制会话 IPC，并保证同一时间只有一个会话。

4. `record/record.html` 与 `record/record.js`
   - 实现待录制、倒计时、录制、暂停、预览和保存进度界面。
   - 获取显示器桌面流，把选区裁剪到隐藏 Canvas。
   - 使用 `canvas.captureStream(frameRate)` 和 MediaRecorder 生成 WebM 分片。
   - 串行发送分片，停止后等待所有分片完成写入。

5. `preload-record.js`
   - 暴露初始化、分片写入、完成录制、保存 MP4、重新录制、边框状态和关闭接口。
   - 不向渲染进程暴露 Node.js 或任意文件系统访问。

6. `record/frame.html`
   - 只渲染边框，不包含业务逻辑和用户输入。
   - 通过主进程消息切换蓝、红、黄三种状态。

7. `main/services/recording-service.js`
   - 管理单个录制会话、临时目录和 WebM 写入队列。
   - 完成文件后返回受控的本地预览 URL。
   - 调用 FFmpeg 转码并解析进度。
   - 负责取消、重新录制、窗口关闭和应用退出时的清理。

8. `record/recording-utils.js`
   - 提供纯函数：选区规范化、显示器内坐标换算、偶数输出尺寸、录制状态转换和 FFmpeg 参数构建。
   - 纯函数由 Node 测试直接覆盖。

### 数据流

```text
截图选区
  -> capture:start-region-recording(selectionBounds)
  -> 主进程验证并解析显示器/sourceId
  -> record:init(sourceId, displayBounds, selectionBounds, frameRate)
  -> getUserMedia 获取完整显示器流
  -> Canvas 按比例裁剪选区
  -> MediaRecorder 每秒产生 WebM 分片
  -> recording-service 串行写入临时文件
  -> 停止后预览临时 WebM
  -> 用户选择 MP4 路径
  -> FFmpeg 转码
  -> 保存 H.264 MP4
```

## 坐标与画面处理

截图选区使用 Electron 的 DIP 屏幕坐标。桌面视频流通常使用物理像素，因此不得直接把 DIP 坐标传给 Canvas。

录制初始化数据包含：

```js
{
  sourceId,
  displayBounds: { x, y, width, height },
  selectionBounds: { x, y, width, height },
  frameRate
}
```

视频元数据就绪后，按实际流尺寸计算比例：

```text
scaleX = videoWidth / displayBounds.width
scaleY = videoHeight / displayBounds.height
sourceX = (selectionBounds.x - displayBounds.x) * scaleX
sourceY = (selectionBounds.y - displayBounds.y) * scaleY
sourceWidth = selectionBounds.width * scaleX
sourceHeight = selectionBounds.height * scaleY
```

Canvas 输出宽高向下调整为大于等于 16 的偶数，保证 H.264 `yuv420p` 可编码。计算同时支持高 DPI、显示器负坐标和非主显示器。

选区必须完全位于同一个显示器内。截图窗口本身按显示器创建，因此正常交互不会产生跨显示器选区；主进程仍会执行边界裁剪与最小尺寸校验。

## 录制状态机

允许的状态为：

```text
idle -> countdown -> recording <-> paused -> preview -> saving -> saved
  |         |            |            |         |         |
  +---------+------------+------------+---------+--------> cancelled
preview -> countdown  // 重新录制
saving -> preview      // 取消保存对话框或转码失败
```

非法状态转换被忽略并记录日志，避免重复点击导致多个 MediaRecorder、多个写入流或重复 FFmpeg 进程。

计时使用单调递增的时间差，并扣除暂停区间；不使用定时器触发次数推算时长。

## 临时文件与资源生命周期

- 每次录制尝试创建独立的会话目录：`userData/temp/recordings/<session-id>/`。
- MediaRecorder 使用约 1 秒的 timeslice 产生分片。
- 渲染进程维护一个 Promise 写入链，确保分片按产生顺序到达主进程。
- 主进程把分片追加到 `capture.webm`，停止时等待文件句柄完成刷新和关闭。
- 重新录制先删除上一次临时文件，再创建新的尝试目录。
- 关闭、取消、保存成功和应用退出时清理会话目录。
- 转码失败时保留当前临时 WebM，用户仍可预览并重试保存；关闭预览后再清理。

## MP4 转码与打包

项目新增 `ffmpeg-static` 运行时依赖，并在 Electron Builder 配置中解包其 Windows x64 可执行文件。开发环境与打包环境都通过统一的路径解析函数获得 FFmpeg，不能依赖系统 PATH。

基础转码参数为：

```text
-i <input.webm>
-an
-c:v libx264
-preset veryfast
-crf 23
-pix_fmt yuv420p
-movflags +faststart
-y <output.mp4>
```

`-an` 明确移除所有音轨。`+faststart` 把 MP4 元数据移动到文件开头，便于网页和聊天软件快速播放。输出文件扩展名固定为 `.mp4`。

保存流程先显示系统保存对话框；用户取消时返回预览，不启动 FFmpeg。转码时从 FFmpeg stderr 的 `time=` 字段计算近似进度。转码失败时删除不完整的目标 MP4，保留预览和临时源文件。

## 配置

复用现有 `record.frameRate` 和 `record.saveDirectory`：

- FPS 仅接受 `5 / 16 / 24 / 30 / 60`，无效值回退到 24。
- 保存对话框默认目录使用 `record.saveDirectory`，未设置时使用系统视频目录。
- 本次不新增声音开关、输出格式选择或质量档位。

## 错误处理

- 无法解析显示器采集源：关闭录制 UI，并向用户显示“无法获取当前显示器画面”。
- `getUserMedia` 失败：停止已有轨道、清理会话并显示具体失败原因。
- Canvas 或 MediaRecorder 初始化失败：不进入倒计时，允许关闭后重新截图选择。
- 分片写入失败：立即停止录制，保留错误日志并清理不可用的临时文件。
- FFmpeg 不存在：预览保持可用，保存按钮显示“MP4 编码组件不可用”。
- FFmpeg 转码失败：删除不完整 MP4，返回预览并允许再次保存。
- 录制窗口异常关闭：主进程停止媒体会话、终止活动转码进程并清理临时资源。

## 测试策略

### 自动化测试

- `record/recording-utils.js`
  - 规范化选区并拒绝小于 16 像素的矩形。
  - 高 DPI 下把 DIP 选区映射到视频像素。
  - 正确处理负坐标副屏。
  - 输出宽高始终为偶数。
  - FPS 白名单与默认值。
  - 状态机只接受允许的转换。
  - FFmpeg 参数包含 H.264、无音频、`yuv420p` 和 `faststart`。

- `main/services/recording-service.js`
  - 分片按顺序写入。
  - 完成、取消、重录和应用退出时按规则清理。
  - 转码失败删除不完整输出并保留预览源。
  - 使用注入的进程启动器测试 FFmpeg 调用，不在单元测试中启动真实编码。

- 静态接线测试
  - 截图工具栏包含区域录制按钮。
  - 预加载脚本只暴露约定 IPC。
  - 打包配置包含录制页面和 FFmpeg 二进制。

### 手工验证

- 主显示器、负坐标副屏和不同缩放比例显示器。
- 5、16、24、30、60 FPS。
- 倒计时取消、暂停/继续、停止、取消和重新录制。
- 预览播放、拖动进度、取消保存、转码失败后重试。
- 较长录制期间内存保持稳定，临时文件持续增长。
- portable 构建中无需系统 FFmpeg 即可保存 MP4。
- 使用 `ffprobe` 确认输出为 H.264、无音轨、像素格式兼容且尺寸与选区一致。

## 非目标

本次不实现以下内容：

- GIF、WebP 或 WebM 用户输出。
- 麦克风、系统声音或音频混合。
- 首尾裁剪、删除中间片段、倍速、滤镜或画质档位。
- 录制中标注、鼠标点击动画、键盘提示或摄像头画中画。
- 跨显示器选区、录制区域移动或录制过程中调整尺寸。
- 自动上传、分享或复制视频文件到剪贴板。

## 验收标准

1. 截图完成选区后可以从工具栏进入区域录制。
2. 录制前显示蓝框和 3 秒倒计时；录制、暂停状态颜色正确。
3. 录制内容只包含选区实时画面，不包含截图遮罩、边框或控制条。
4. 暂停时间不计入视频和显示时长。
5. 停止后能够预览、重新录制或保存。
6. 保存结果固定为可播放的 H.264 MP4，且没有音轨。
7. 高 DPI 和负坐标副屏下，输出区域与用户选区一致。
8. 取消、关闭、重录和保存完成后不存在遗留录制进程或临时文件。
9. Windows portable 构建包含 FFmpeg，目标机器无需预装额外软件。
